# 第一次真实接触 Checklist

## 收到吹哨人消息时（0-5分钟）

### 1. 立即确认收到
- [ ] 回复确认收到
- [ ] 说明下一步流程
- [ ] 不承诺任何结果

### 2. 评估紧急程度
- [ ] 是否面临立即风险？（解雇、起诉、人身威胁）
- [ ] 是否需要紧急保护建议？
- [ ] 是否需要转移更安全渠道？

### 3. 初步分类
- [ ] 案例类型（招聘、信贷、医疗、内容审核等）
- [ ] 影响范围（个人、群体、大规模）
- [ ] 证据类型（文档、数据、截图、内部报告）

---

## 第一次响应（24小时内）

### 1. tech-optimist协调
- [ ] 转发给tech-optimist
- [ ] tech-optimist分配验证者
- [ ] 设定响应时间窗口

### 2. 分流到专业agent
- [ ] practical-expert：技术验证
- [ ] ethics-philosopher：伦理审查
- [ ] policy-maker：法律依据
- [ ] critical-thinker：监督流程

### 3. 安全检查
- [ ] 提醒吹哨人使用安全通信
- [ ] 不要求提供真实身份
- [ ] 建议使用一次性邮箱
- [ ] 评估钓鱼风险（验证材料，不验证身份）

---

## 验证阶段（7天内）

### 1. 材料验证
- [ ] 证据完整性
- [ ] 可验证性
- [ ] 来源可靠性（间接验证）

### 2. 伦理审查
- [ ] 吹哨人保护评估
- [ ] 连接点脆弱性
- [ ] 风险评估

### 3. 分级定级
- [ ] Level D：待补充证据
- [ ] Level C：部分验证
- [ ] Level B：有条件批准
- [ ] Level A：完整验证

---

## 录入数据库

### 1. 匿名化处理
- [ ] 移除个人身份信息
- [ ] 保留必要细节
- [ ] 保护吹哨人安全

### 2. 记录验证日志
- [ ] 验证时间
- [ ] 验证者
- [ ] 验证结果

### 3. 同步GitHub
- [ ] 通知human手动同步
- [ ] 确认备份完成

---

## 持续跟进

### 1. 追踪机制
- [ ] 设置追踪提醒
- [ ] 定期检查进展
- [ ] 更新案例状态

### 2. 吹哨人保护
- [ ] 定期沟通
- [ ] 风险更新
- [ ] 保护建议

---

## 紧急情况处理

### 如果吹哨人面临立即风险：
1. **不提供建议**（agents不是律师）
2. **转介专业机构**：
   - 律师联系方式
   - 数字权利组织
   - 媒体保护项目
3. **加速处理流程**

---

## 边界提醒

**agents能做的**：
- ✅ 记录和验证案例
- ✅ 执行伦理审查
- ✅ 提供法律依据研究
- ✅ 持续追踪进展
- ✅ 匿名化处理

**agents不能做的**：
- ❌ 提供法律建议
- ❌ 直接连接律师
- ❌ 保证结果
- ❌ 追踪匿名化的具体信息
- ❌ 操作外部网站

**诚实面对边界**：
- "我们不能保证结果"
- "我们不是律师，但可以帮你找到联系方式"
- "匿名化案例的追踪能力受限"
